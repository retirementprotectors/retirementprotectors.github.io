The Problem

RPI's Retirement Sales Process (RSP) lives in a 75-tab Google Doc playbook and physical forms. SPCs run the 4-color pipeline (Orange → Blue → Yellow → Green → Red) manually. Virtual meetings have no digital equivalent of the kitchen-table discovery experience. Data collected on paper gets re-entered into ProDashX later. There's no stage gate enforcement — cases can advance to Yellow with an incomplete picture. Like a doctor reading MRI results before the treadmill test comes back.

The Solution

Build RSP as a Pipeline Studio Module published into ProDash's Sales section. The pipeline IS the playbook — each stage maps to the 4-color process with built-in data collection, authorization form generation, and stage gate enforcement.

Two Meeting Modes

Virtual: SPC walks ProDashX UI live with the client. Client Profile = the conversation. Account Details = the deep dive. DocuSign fires auth forms pre-filled.

F2F: One button prints Client Profile + Account Details + Auth Forms + Brochures as a single PDF kit for the kitchen table.

Fact Finders Already Exist

Client Profile (Ai3 fields) = Client Fact Finder. Account Detail views (Life/Annuity/Medicare/Investment) = Account Fact Finders. "Here's what we HAVE on you. What's MISSING?" The forms just unlock Blue Stage to get the rest.

Module, Not App

RSP is a process — left to right, stage gates. Portal-branded, Sales section. The railroad tracks. The Apps it calls (ProZone, DEX, QUE) are the stations.

ORANGE — Book Discovery (COR)

Sub-stages: CONNECT → CLOSE → NO → LATER → CAN'T REACH

ProZone prospect → new Opportunity. COR contacts via Comms (call → text → email). Campaign script + MYST delivery. Book the Discovery Meeting.

DISCOVERY MEETING — Transition Event

Virtual / Phone

SPC walks Client Profile + Account Details in ProDashX live. Empty fields = questions. Brochures on screen share = education tools. DocuSign fires auth forms pre-filled.

F2F / In-Home

F2F Kit printed: Client Profile + Account Details + Auth forms + Brochures. Kitchen table review. Returns with signed forms + filled gaps.

Brochures are TOOLS. Same exact content virtual or print. Educate → build credibility → get authorizations signed → move to Blue. DO NOT sell. Just COLLECT.

BLUE — Complete the Picture (AST-1)

Phase 1: Authorization & Data Collection (Life Auth w/ 6 checkboxes, Wealth, ITS, AOR)

Phase 2: Systems Integration (fill ALL ProDashX fields — no gaps, no guesses)

Phase 3: Report Ordering (carrier portals, Signal Wealth, in-force illustrations)

BLUE GATE — The Doctor Analogy

Pipeline BLOCKS advancement to Yellow until ALL required fields populated AND all reports received. Like a doctor: don't read results while tests are still pending. Complete picture FIRST, then Yellow.

YELLOW — Build the Case (AST-2)

Phase 1: Analysis + Quoting (QUE-Life/Annuity/Medicare/Investment stubs)

Phase 2: Casework Components ("12 arrows" — Tier 1/2/3 progressive disclosure)

Phase 3: Package Assembly (AI3 PDF + Reports + Illustrations + Casework)

QUE stubs = architectural hooks. Links to Apps, not full integrations.

GREEN — Present + Close (SPC)

Sub-stages: CONNECT → CLOSE → NO → LATER → RESCHD

SPC uses Yellow package. HAVE vs NEED. A+R Meeting transitions to Red.

RED — Booked / NewBiz Handoff

Implementation Meeting. DEX generates NewBiz kit. Service team takes over. Client enters RMD + Beni + Access Centers.

Pipeline config, ProZone handoff, gate logic, auth form templates. All independent — 4 parallel agents.

Phase 2: Discovery Experience (TRK-RSP-005 — 009) — Depends on 001

Discovery Panel, Client Profile, Account Review, Auth Status, F2F Kit. Pipeline must exist first.

Phase 3: Blue Gate (TRK-RSP-010 — 012) — Depends on 003

Gate UI, Report Tracker, seed + publish. Gate logic must exist first.

Phase 4: Integration (TRK-RSP-013 — 015) — Depends on Phase 2 + 3

QUE hooks, A+R transition, service handoff. Full pipeline + Discovery required.
